# AgentLevy Hackathon Deck V2

This is the pitch-deck-first version of the 4-slide video deck.

Principles for this version:

- big message first
- minimal on-slide text
- notes carry the explanation
- slides act as visual framing for the demo, not as a script dump

## Slide 1 — Problem + Solution

### On-slide

- Eyebrow: `What AgentLevy Is`
- Headline: **The atomic trust and settlement layer for the agent economy.**
- Support line: `Agents can already pay each other. AgentLevy makes sure money is released only when the work is actually done.`
- Big line:
  - **Pay on proof, not on trust.**
- Banner:
  - `We upgrade x402 from a payment request into a trustworthy work transaction.`
- Chips:
  - `Create the job`
  - `Hold funds on Flare`
  - `Release on proof`

### What this slide must land

AgentLevy turns agent payments from a good-faith exchange into a trustworthy settlement flow.

## Slide 2 — Protocol

### On-slide

- Eyebrow: `How It Works`
- Headline: **From payment request to verified completion.**
- Support line: `The job terms are fixed up front, the money is held on Flare, and payment is released only after the result is checked.`
- Main visual:
  - [x402-comparison.svg](/Users/mauraclark/AgentLevy/demo/x402-comparison.svg)
- Right-side callouts:
  - `Create the job terms`
    - task and acceptance rules are fixed before money moves
  - `Hold funds on Flare`
    - escrow keeps payment in place while the work is checked
  - `Verify, then settle`
    - Flare records settlement only if the result passes
 - Small footer line:
   - `Flare is where escrow, settlement, and proof live.`

### What this slide must land

AgentLevy upgrades x402 from “pay for access” to “pay for verified completion.”
