## Visual Takeaways From The Existing Deck

### Mood

The deck reads as:

- formal
- institutional
- policy-adjacent
- serious rather than playful
- dark and high-contrast

That is a good fit for AgentLevy's current trust-layer story.

### Core visual direction

The current deck uses:

- a deep navy background
- bright cyan accents
- bright green for success or shipped-state emphasis
- large white headlines
- muted steel-blue supporting text
- orange banners for urgency or key emphasis
- purple as a secondary accent for technical differentiation
- simple robot iconography

### Replace the stale cover tagline

The current deck cover contains a time-sensitive layoffs/news line that is now outdated.

For the next version of the deck, keep the **orange banner treatment** but replace the text with a timeless protocol line.

Recommended primary replacement:

**The missing trust layer for the agentic economy.**

Recommended alternatives:

- **Agents can already pay. They still cannot prove work before release.**
- **x402 requests payment. AgentLevy verifies completion before value moves.**
- **Trust, escrow, verification, and proof for agent-to-agent commerce.**

### Logo / mark treatment

- a large stacked `Agent / Levy / Protocol` wordmark
- `Agent` in white
- `Levy` in cyan

- a robot icon used both as a large hero element and a small utility mark

For the website and video overlays, that same hierarchy should be preserved:

## Editorial Style To Reuse

### Headline style

The deck favors:

- short uppercase section labels
- bold declarative slide titles
- compressed one-sentence conclusions

Examples of reusable structure:

- `THE PROBLEM`
- `HOW IT WORKS`
- `THE PROTOCOL STACK`
- `WHAT WE'RE BUILDING THIS WEEKEND`

For the current project, keep that same style but use the new product story:

- `THE TRUST GAP`
- `HOW AGENTLEVY WORKS`
- `WHY 0G + FLARE`
- `THE LIVE DEMO`
- `THE ATOMIC TRUST LAYER`

### Copy style

The writing style is:

- direct
- compressed
- persuasive
- easy to scan

It works best when each section has:

- one clear claim
- one short explanation
- one diagram or proof artifact

## What To Reuse

Reuse these design patterns:

- dark hero sections
- large high-contrast typography
- cyan emphasis for primary branded terms
- orange callouts for urgency, contrast, or a short protocol thesis
- strong left-aligned storytelling blocks
- architecture sections with numbered steps or stack layers

## What To Avoid

Avoid:

- default white startup-site styling
- soft pastel palettes
- generic gradient-only SaaS design
- overly playful agent avatars
- purple-heavy design drift
